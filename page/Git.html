TortoiseGit使い方

既存のリポジトリのgit urlを指定してローカル環境に複製する【git clone】

「git URL」を確認。例としてherokuのものを使用

環境を作成したいディレクトリで右クリック「Git clone」を選択

URLに「git URL」を設定。Directoryに複製するパスを指定。OKをクリック。

「close」を選択

Directoryに指定したディレクトリにgitの内容が複製されていることを確認。隠しファイルを表示にすると「.git」が作成されているが、これはGitの情報を格納する隠しディレクトリらしい。複製を削除する場合はこのディレクトリごと削除すれば良いらしい。

ファイルを追加する。【git add/commit/push】

適当なファイルを作成する

ファイルを右クリックファイルをadd(作業ディレクトリ内の変更をステージングエリアに追加)する

OKをクリックする

ファイルを右クリック。Git Commit(変更を確定)する

メッセージを記載。Commitをクリックする。コミットする対象が合っているかどうかを見ておく。

closeをクリックする。

上記の手順でローカル環境からローカルリポジトリ(自分のパソコン・マシン内に存在するリポジトリ)に追加した状態になる。

試しに「https://toriisan-php-001.herokuapp.com/test.html」にアクセスしてもファイルが見つからないとなる。

追加したファイルのディレクトリを右クリックしてpush(ローカルリポジトリコンテンツをリモートリポジトリにアップロード)する。

OKを選択する。

closeを選択する。

再度「https://toriisan-php-001.herokuapp.com/test.html」にアクセス。リモートリポジトリにアップロードされていることが確認出来る。

ファイルを修正する。

[サーバー]「sudo yum update -y」でパッケージの最新化

[サーバー]「sudo yum -y install httpd」でhttpサーバーをインストール

[サーバー]「sudo yum install git -y」でGitをインストール

[サーバー]「sudo service httpd start」でweb サーバーを起動

[サーバー]「sudo chkconfig httpd on」でサーバを再起動した時に自動的に httpd も起動

[サーバー]「sudo mkdir -p /var/lib/git」でgitのディレクトリを作成。「-p」オプションで間のディレクトリが無い場合は作成。

[サーバー]「cd /var/lib/git」でディレクトリに移動

[サーバー]「sudo mkdir my-repo.git」でGitディレクトリを作成。

[サーバー]「cd my-repo.git」でGitディレクトリへ移動。

[サーバー]「sudo chown -R ec2-user:ec2-user /var/lib/git」で「ec2-user」に「/var/lib/git」の権限を付与。sudoを使わなくて良くなる。

[サーバー]「sudo vi /etc/httpd/conf.d/git.conf」でGitのリポジトリに対してHTTP経由でのアクセスを設定

SetEnv GIT_PROJECT_ROOT /var/lib/git
SetEnv GIT_HTTP_EXPORT_ALL
ScriptAlias /git/ /usr/libexec/git-core/git-http-backend/

<Location /git>
    Options ExecCGI
    AuthName "Git for HTTP"
    AuthType Basic
    AuthUserFile /etc/httpd/conf/.htpasswd
    Require valid-user
</Location>

[サーバー]「sudo systemctl restart httpd」でApacheの再起動

[サーバー]「sudo chgrp -R apache /var/lib/git/my-repo.git」でディレクトリの所有者をapacheに変更。-Rはリカーシブ（再帰的）な操作を行うためのオプションで、ディレクトリ内のすべてのファイルやサブディレクトリに変更を適用。

[サーバー]「sudo htpasswd -c /etc/httpd/conf/.htpasswd git」でApacheの基本認証（Basic Authentication）で使用されるユーザー名とパスワードを追加

[ec2-user@************ my-repo.git]$ sudo htpasswd -c /etc/httpd/conf/.htpasswd git
